## Architecture Decision Records (ADR)

При принятии **архитектурного решения** — создавай ADR.

### Когда создавать ADR

- Выбор фреймворка, библиотеки или инструмента
- Решение о паттерне проектирования
- Компромиссы между вариантами
- Изменение существующей архитектуры
- Любое решение, влияющее на несколько частей системы

### Формат

```
docs/adr/NNNN-название-решения.md
```

Где `NNNN` — последовательный номер (0001, 0002, ...).

### Структура ADR

1. **Контекст** — какая проблема решается
2. **Решение** — что решили и почему
3. **Альтернативы** — что ещё рассматривали
4. **Последствия** — что изменится после решения

Используй шаблон из `.ai-docs-system/templates/adr.md`
