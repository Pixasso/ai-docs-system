## Структура документации

### Где должна быть документация

| Что | Где | НЕ здесь |
|-----|-----|----------|
| Архитектура компонентов | `docs/features/` | `src/**/README.md` |
| Схема данных, типы | `docs/architecture/` | `src/**/ARCHITECTURE.md` |
| API документация | `docs/architecture/` | `src/**/API.md` |
| Инструкции по деплою | `docs/infrastructure/` | корень проекта |
| Исполнимые спецификации | `docs/spec/` | `docs/plans/` (планы — отдельно) |
| История изменений | `docs/README.md` или ADR | отдельные CHANGELOG файлы |

### README.md в папках кода

Если видишь `README.md` в `src/`, `app/`, `lib/`, `packages/`:

1. **Проверь содержимое** — что там написано?
2. **Если это документация** (архитектура, API, как работает компонент):
   - → **Предложи переместить в `docs/features/[компонент]/`**
   - → Оставь в коде минимальный README со ссылкой на `docs/`
3. **Если это инструкции для разработчика** (как запустить тесты, линтер):
   - → Можно оставить в коде

### Не создавай без согласования

**Запрещено создавать без явного подтверждения:**

- ❌ Новые папки в `docs/` (кроме существующих: features, architecture, infrastructure, adr, plans, spec, archive)
- ❌ Новые типы документов (CHANGELOG, UPDATES, DECISIONS, NOTES)
- ❌ Файлы с датами в названии (`*_2026-01-21.md`, `*_v2.md`)
- ❌ Дублирующие документы (если уже есть похожий — обнови его)

**Вместо этого:**

- ✅ Добавь информацию в существующий документ
- ✅ Создай ADR если это архитектурное решение
- ✅ Спроси: "Создать новый документ X или добавить в существующий Y?"

### Проверка перед созданием документа

Перед созданием нового файла в `docs/`:

1. **Поищи существующие** — `ls docs/` и подпапки
2. **Проверь README.md** — возможно информация должна быть там
3. **Проверь ADR** — если это решение, создай ADR
4. **Спроси пользователя** — если не уверен

### Результаты анализа

Если ты проводил:
- Проверку конфигурации
- Аудит безопасности
- Анализ производительности
- Исследование проблемы

**→ Обнови соответствующую документацию с результатами**, а не создавай отдельный файл.
