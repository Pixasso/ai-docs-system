# Architecture Decision Records (ADR)

> **Status:** current  
> **Last verified:** 2026-02-03  
> **Owner:** @Pixasso

Задокументированные архитектурные решения для проекта.

## Что такое ADR?

Architecture Decision Record фиксирует важное архитектурное решение вместе с его контекстом и последствиями.

## Когда создавать ADR

- Выбор фреймворка, библиотеки или инструмента
- Решение о паттерне проектирования
- Компромиссы между вариантами
- Изменение существующей архитектуры
- Любое решение, влияющее на несколько частей системы

## Шаблон

```markdown
# ADR-NNNN: Название решения

> **Status:** proposed | accepted | deprecated | superseded  
> **Date:** YYYY-MM-DD  
> **Author:** @Pixasso  
> **Supersedes:** ADR-XXXX (если применимо)

## Контекст

[Опишите ситуацию и проблему, которую нужно решить.
Какие факторы действуют? Какие есть ограничения?]

## Решение

[Опишите принятое решение.
Будьте конкретны о том, что было выбрано и почему.]

## Рассмотренные альтернативы

### Вариант A: [Название]
- Плюсы: ...
- Минусы: ...

### Вариант B: [Название]
- Плюсы: ...
- Минусы: ...

## Последствия

### Положительные
- [Хороший результат 1]
- [Хороший результат 2]

### Отрицательные
- [Компромисс 1]
- [Риск 1]

## Ссылки

- [Ссылка на релевантную документацию]

```

- `0002-adopt-typescript.md`
