# Исполнимые спецификации

> **Status:** draft  
> **Last verified:** YYYY-MM-DD HH:MM  
> **Owner:** @username

Детальные спецификации для реализации планов из `docs/plans/`.

## Связь Plan ↔ Spec

Каждый план (`docs/plans/phase-*.md`) должен ссылаться на свою спецификацию:

```markdown
Execution spec: docs/spec/<имя-спецификации>.md
```

## Шаблон спецификации

```markdown
# Spec: [Название]

> **Status:** draft | ready | implemented  
> **Last verified:** YYYY-MM-DD HH:MM  
> **Owner:** @username

Связанный план: docs/plans/<plan-name>.md

## Цель

[Что должно быть реализовано]

## Требования

### Функциональные
- [ ] Требование 1
- [ ] Требование 2

### Нефункциональные
- [ ] Производительность: ...
- [ ] Безопасность: ...

## Этапы реализации

1. **Этап 1:** [Описание]
   - [ ] Задача 1.1
   - [ ] Задача 1.2

2. **Этап 2:** [Описание]
   - [ ] Задача 2.1

## Риски и митигации

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| ... | высокая/средняя/низкая | ... |

## Чек-лист готовности

- [ ] Код написан
- [ ] Тесты пройдены
- [ ] Документация обновлена
- [ ] Code review пройден
- [ ] Задеплоено в staging
```

## Примечания

- Spec — это **исполнимый документ** с чек-листами
- Plan — это **стратегический документ** (цели, сроки, зависимости)
- При изменении плана — проверь и обнови связанную спецификацию
